// =========================================================== Description
== Description

// tag::body[]
{manpurpose}

Some actions (e.g. xref:aerospace-layout.adoc[`layout floating`/`layout tiling`] or
xref:aerospace-center-floating-window.adoc[`center-floating-window`]) are otherwise completely silent,
which is especially noticeable when they're triggered by a hotkey rather than the CLI (a hotkey has no
terminal to print errors or results to). When notifications are enabled, a tiny "AppName : Action" toast
briefly appears on screen instead.

This setting is a runtime toggle, not a config option - it resets to enabled every time AeroSpace starts.
